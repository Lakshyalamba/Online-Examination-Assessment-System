# Admin Dashboard Inspiration Prompt

## Page Purpose

Provide operational oversight across users, exams, activity, and system health.

## Key Visual Hierarchy

- top metrics row
- activity and audit previews
- role distribution or usage chart
- pending issues or bottleneck panels

## Target Aesthetic

Confident, governance-oriented, and efficient.

## Important Sections And Components

- operational summary cards
- recent system activity panel
- role distribution chart
- pending review or attention-needed modules

## Interaction Feel

Analytical and structured rather than promotional.

## Layout Constraints

- dashboard should feel denser than student pages
- charts should support, not dominate, the page

## Consistency Rules From `design.md`

- shared metric-card language
- restrained color semantics
- same table and sidebar patterns as other internal areas

Avoid: overdecorated BI dashboard look, dark theme, unnecessary 3D widgets
