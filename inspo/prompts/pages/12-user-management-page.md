# User Management Page Inspiration Prompt

## Page Purpose

Support account administration with clear role visibility, status control, and low-friction management patterns.

## Key Visual Hierarchy

- page title and create-user action
- searchable user table
- filters by role and status
- detail drawer or edit modal

## Target Aesthetic

Table-driven, clear, and operationally steady.

## Important Sections And Components

- table with avatar or initials, email, role, status, last activity
- filters and search
- create or invite action
- role edit or deactivate controls

## Interaction Feel

Controlled and admin-focused, with minimal ambiguity around action outcomes.

## Layout Constraints

- keep table readable at dense row counts
- drawer or modal should feel like part of the same system shell

## Consistency Rules From `design.md`

- use standard role badges
- keep action placement and table styling consistent
- use danger colors only for deactivation or destructive controls
