# Cards, Tables, Forms, And Modals Inspiration Prompt

## Component Purpose

Define the visual standards for the highest-frequency UI primitives in the system.

## Key Visual Hierarchy

- summary cards
- data tables
- form sections
- modal confirmations
- warning and destructive states

## Target Aesthetic

Operational, balanced, and professional.

## Important Sections And Components

- metric and detail cards
- dense management table
- form field groups with helper and error text
- confirmation and destructive modals

## Interaction Feel

Predictable, high-contrast, and easy to scan repeatedly.

## Layout Constraints

- show primitives side by side on one coherent board
- keep spacing and shadows disciplined

## Consistency Rules From `design.md`

- card structure should match dashboards
- tables and forms must share the same border and spacing language
- modal hierarchy must clearly separate primary and cancel actions

Avoid: oversaturated UI kits, glossy enterprise chrome, dark theme, inconsistent spacing
