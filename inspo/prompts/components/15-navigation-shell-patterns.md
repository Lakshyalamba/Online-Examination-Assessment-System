# Navigation Shell Patterns Inspiration Prompt

## Component Purpose

Define the structural navigation shell shared across Admin, Examiner, and Student experiences.

## Key Visual Hierarchy

- sidebar
- top header
- breadcrumbs
- content canvas
- profile and utility actions

## Target Aesthetic

Stable, compact, trustworthy, and role-aware.

## Important Sections And Components

- desktop sidebar
- collapsed tablet state
- top navigation with page title and actions
- user menu area
- notification or status slot

## Interaction Feel

Reliable and consistent, with strong wayfinding and minimal noise.

## Layout Constraints

- show at least one desktop and one compact responsive state
- keep the navigation shell lighter than enterprise admin software bloat

## Consistency Rules From `design.md`

- navy structural band for shell
- teal active-state indicator
- consistent header spacing and icon treatment

## Suggested Image Prompt

Use case: `ui-mockup`  
Asset type: navigation shell reference  
Primary request: polished navigation shell board for a role-based examination management platform, showing navy sidebar, top header, breadcrumb pattern, profile menu, active navigation states in teal, content canvas examples, responsive collapse behavior, light internal product aesthetic, clean and implementation-friendly  
Avoid: mega-menu complexity, dark dashboard chrome, inconsistent iconography, playful consumer-nav styling
